# Product Requirements Document (PRD): Selva

## 1. Product Vision & Concept

**Selva** is a gamified, multi-agent business orchestration platform. It allows users to manage their entire digital enterprise operations as an "Auto Chess" style RPG. Users act as the "Tactician," using a physical gamepad to navigate a 2D virtual office, draft AI agents, assign them to departments, and physically walk up to "Review Stations" to manually approve critical business actions (emails, code deployments, CRM updates) generated by the autonomous swarms.

## 2. Licensing Strategy & Core Tech Stack

Selva is released under the **AGPL-3.0** license. Any network-accessible deployment of this software (including SaaS hosting) triggers the obligation to disclose the complete corresponding source code to its users. The architecture utilizes the following stack:

* **Spatial Frontend: Custom Phaser 3 + Colyseus + React.** A purpose-built 2D virtual office environment using Phaser 3 (game engine), Colyseus (WebSocket server), and React. This gives us full ownership of the 16-bit RPG environment.
* **Central Orchestration Gateway: OpenClaw.** OpenClaw serves as the persistent background daemon, managing API connections, scheduled behaviors, and localized memory.


* **Execution Engine: LangGraph & Ruflo.** The heavy parallel execution pipelines are built using **LangGraph** (a framework designed for cyclic, multi-agent workflows) and **Ruflo** (an enterprise AI orchestration platform designed for deploying specialized swarms). These handle isolated Git worktrees and granular tool permissioning natively.

## 3. Core Features & System Requirements

### 3.1. The RPG Spatial Matrix (SkyOffice Implementation)

* **Controller Navigation:** The web client will utilize the HTML5 Gamepad API (`navigator.getGamepads()`) to map character movement to analog sticks and interaction to face buttons.
* **Embedded Dashboards:** We will modify SkyOffice to support interactive React UI components embedded directly over the Phaser 3 canvas, allowing users to view dynamic Agent Kanban boards and logs without leaving the game window.
* **Cross-Origin Communication:** Embedded review iframes will securely communicate with the main SkyOffice game loop using the `window.postMessage()` API, allowing gamepad inputs to trigger approvals inside the dashboards.

### 3.2. Central Nervous System & Memory (OpenClaw Implementation)

* **Proactive Heartbeats:** The OpenClaw gateway must run a 30-minute cron job ("Heartbeat") allowing agents to wake up, scrape CRM data, parse code PRs, and triage support tickets without human prompting.


* **Markdown Persistence:** All long-term enterprise knowledge, standard operating procedures, and daily logs must be written to human-readable Markdown files (`MEMORY.md`, `SOUL.md`).


* **Model Agnosticism:** The system must support flexible routing between local models (Ollama) for sensitive data and hosted models (Claude 3.5 Sonnet / Opus 4.6) for complex reasoning.



### 3.3. Autonomous Execution & Synergies (LangGraph / Ruflo Implementation)

* **Parallel Worktrees:** When a coding or complex data task is triggered, the system must spawn isolated Git worktrees to ensure the main operational branch is protected from agent hallucinations.
* **Swarm Roles:** The system must support drafting specialized agents (e.g., `planner`, `coder`, `reviewer`, `researcher`) that can operate concurrently.
* **Auto Chess Synergies:** The orchestration logic must calculate efficiency multipliers based on agent placement. For example, assigning a "Research Agent" and a "Build Agent" to the same virtual room in SkyOffice unlocks a "Surgical DevOps" speed modifier.

### 3.4. Human-in-the-Loop (HITL) Quality Gates

* **Zero-Surprise Architecture:** All destructive or outbound actions (e.g., executing `bash` scripts, dispatching emails, committing code) must be strictly defined in a JSON permission matrix as `"ask"`.


* **LangGraph Interrupts:** The execution pipeline will utilize LangGraph's `interrupt()` function to pause an agent's state immediately before an outbound action.
* **The Review Station Gameplay Loop:**
1. An agent hits an `interrupt()` block.
2. An alert icon appears over the agent's avatar in SkyOffice.
3. The Tactician (User) walks their avatar to the agent.
4. A UI modal pops up showing the drafted email/code diff.
5. The user presses the "A" button on their gamepad to send the `PermissionResultAllow` webhook, or "B" to send a `PermissionResultDeny` with feedback.



## 4. Operational Workflows (Use Cases)

### Scenario A: The Morning Triage Wave

The game begins at 9:00 AM. OpenClaw's heartbeat triggers a wave of "enemies" (new support tickets and GitHub issues). The drafted CRM sub-agents automatically begin fetching context and drafting responses. The user walks through the virtual office, approaching each agent's desk. The user rapidly presses 'A' to approve perfect email drafts, clearing the enemy wave and earning "Compute Tokens" for the day's budget.

### Scenario B: Feature Deployment Boss Fight

The user spends Compute Tokens to upgrade their Engineering Swarm to Level 2. The swarm is tasked with integrating a new payment gateway. Ruflo handles the orchestration, spinning up 5 parallel agents working safely in isolated worktrees. When a bug occurs, the swarm enters an automatic QA recovery loop. Once tests pass, the Lead Engineer bot flags the user down in SkyOffice for final human approval on the Pull Request.

## 5. Future Commercialization Strategy

Because Selva is licensed under **AGPL-3.0**, any party that deploys the software over a network must make the complete corresponding source code available to its users. This copyleft requirement protects the open-source community while enabling several commercial models:

* **Dual licensing:** Offer a commercial license for organizations that need to deploy Selva without the AGPL-3.0 source-disclosure obligation (e.g., proprietary SaaS or embedded distributions).
* **Enterprise support & services:** Sell premium support tiers, SLA-backed hosting, and professional services (custom integrations, onboarding, training).
* **Managed cloud offering:** Operate an official hosted version. Because the operator controls the canonical codebase, AGPL-3.0 compliance is straightforward while competing on operational excellence.

Future roadmap items include multi-tenant support, enterprise SSO integration, and a mobile-friendly tactician dashboard.
